__________________Set____________________
- Là 1 interface, là 1 phần của java collection
- Đại diện cho tập hợp các phần tử duy nhất, không cho phép giá trị trùng lặp.
- Set chỉ chứa các phần tử độc nhất
- Không có chỉ mục không thể truy cập bằng getIndex như list
#                                  Set
|                                   | implements
|                                   |
|      HashSet                   LinkedSet                        TreeSet

## HashSet:
 #syntax: Set<E> name = new HashSet<>() or HashSet<E> name = new HashSet<>()
 - Sử dụng data structure: hashTable
 - Kích thước mặt định: capacity: 16, load factor = 0.75, phần tử vượt quá > capacity * load factor, hashSet tự động tăng kích thước để giữ hiệu năng
 - Implements set interface.
 - Cho phép 1 giá trị null
 - Lưu trữ các phần tử duy nhất, không cho phép các phần tử trùng lặp
 - Không duy trùy bất kỳ thứ tự nào của các phần tử. hashSet sử dụng HashMap bên trong, khi in ra nó duyệt theo thứ tụ bucket từ thấp đến cao
 - nhanh - không cần thứ tự
 #Ứng dụng:
  - loại bỏ phần tử trùng lập
  - Tìm kiếm nhanh - kiểm tra giá trị có tồn tại hay khổng -> contains() nhanh hơn list
  - Không quan tâm thứ tự
 #Không nên dùng:
  - Giữ nguyên thứ tự
  - Cần luôn sắp xếp
  - Truy cập theo index

## LinkedHashSet:
 #Syntax: set<E> name = new LinkedHashSet<>() or LinkedHashSet<E> name = new LinkedHashSet<>()
 - Kết hợp chức năng của HashSet với một doubly-linked list 
 - Doubly-linkedList cho phép duy trì thứ tự chèn của các phần tử.
 - Cho phép một giá trị null
 - Duy trì thứ tự ổn định suốt đời - thêm lại phần tử đã xóa không thay đổi vị trí cũ
 - Nhanh - giữ nguyên thứ tự.
 # Dùng khi:
 - Loại bỏ trùng lặp phần tử nhưng vẫn giữ thứ tự phần tử

## TreeSet:
 #Syntax: Set<E> name = new TreeSet<>() or TreeSet<E> name = new TreeSet<>()
 - Là 1 lớp collection 
 - Lưu trữ các phần tử duy nhất, không lặp lại
 - Sắp xếp theo thứ tự 
 - Triển khai interface SortedSet, red-Black Tree (cây tự cân bằng)
 - Không cho phép giá trị trùng lặp và không cho phép giá trị null
 - Sử dụng iterator duyệt theo thứ tự tăng dần hoặc theo comparator cung cấp
 # Dùng khi:
  - Lưu danh sách tên, mã, điểm... cần hiển thị thứ tự
 # Không nên:
  - Dữ liệu quá lớn
  - Cần dữ nguyên thứ tự
 # Constructor:
 - TreeSet(): set<E> name = new TreeSet<>()
 - TreeSet(Comparator): set<E> name = new TreeSet<>(Comparator)
